# Postgres Курс

Учебный проект для изучения работы с PostgreSQL через Python. 
Реализует простую систему управления складами и товарами с интерактивным CLI-интерфейсом.

## О проекте

Это шаблон для изучения PostgreSQL, который демонстрирует:
- Подключение к PostgreSQL через `psycopg3`
- CRUD операции (Create, Read, Update, Delete)
- Интерактивный CLI с автодополнением команд
- Красивый вывод данных в терминале

## Структура проекта

```
postgres-course/
├── src/                      # Исходный код приложения
│   ├── main.py              # Точка входа, основной цикл приложения
│   ├── commands.py          # Определение команд и автодополнение
│   ├── console.py           # Настройка консольного вывода (rich)
│   ├── validators.py        # Валидаторы для пользовательского ввода
│   ├── warehouses.py        # Логика работы со складами
│   └── products.py          # Логика работы с товарами
├── compose.yaml             # Docker Compose для запуска PostgreSQL
├── requirements.txt         # Зависимости Python
├── Makefile                 # Команды для запуска инструментов
├── .pylintrc                # Конфигурация pylint
├── mypy.ini                 # Конфигурация mypy
└── .gitignore               # Игнорируемые файлы для git
```

### Описание модулей

- **main.py** - главный файл с циклом обработки команд, подключением к БД и маршрутизацией команд
- **commands.py** - константы команд и настройка автодополнения через `prompt_toolkit`
- **console.py** - настройка консоли для красивого вывода через библиотеку `rich`
- **validators.py** - валидаторы для проверки пользовательского ввода
- **warehouses.py** - класс `WarehousesHandler` для работы со складами (CRUD операции)
- **products.py** - класс `ProductsHandler` для работы с товарами (CRUD операции)

## Технологии

- **Python 3.13** - язык программирования
- **PostgreSQL 18** - база данных
- **psycopg3** - драйвер для работы с PostgreSQL
- **prompt-toolkit** - интерактивный CLI с автодополнением
- **rich** - красивый вывод в терминале
- **alembic** - миграции

## Инструменты разработчика
### black
Форматтер кода. Запуск:
```shell
black .
```
### pylint
Линтер. Запуск:
```shell
pylint src
```
Игнорировать ошибку на текущей строке можно, добавив
```
# pylint: disable={имя правила}
```
после кода.
### mypy
Статический анализатор. Запуск:
```shell
mypy .
```
Игнорировать ошибку на текущей строке можно, добавив
```
# type: ignore
```
после кода.
## Запуск всех инструментов одной командой
```shell
make all
```
